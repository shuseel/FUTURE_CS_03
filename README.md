# Future Interns - Cyber Security Task 3
## Title
API SECURITY RISK ANALYSIS REPORT

## Objective
* Conduct a read-only security risk analysis of a public API
* Find common security flaws in the API
* Offer precise remedial suggestions

## Tools Used
* Postman 
* Browser Developer Tools
* Manual HTTP analysis

## Scope
* Limited to publicly accessible endpoints of the JSON placeholder API
* Only safe GET and regulated POST requests were carried out

## Methodology
* Documentation review
* Endpoint enumeration
* Authentication verification
* Authorization testing
* Response analysis
* Header inspection

## API Tested
JSON Placeholder Base URL: https://jsonplaceholder.typicode.com

## Risk Identified
* Unauthenticated Access
* Broken Object Level Authorization (BOLA)
* Excessive Data Exposure
* Missing Rate Limiting
* Security Header Deficiencies
